Lanky Standard Library
Table
A quick hashtable implementation; wraps the interpreter hashtable (which is used for storing members). Custom objects can override default hash behavior by implementing their own hash_ method. Note that if you use your own hash_ method, you should also specify your own op_equals_ method.

Prototype Fields
  • count:The number of elements in the table
  • size_:The number of buckets allocated for the table
  • hb_:The binary blob holding a reference to the actual hashtable
Static Methods
  • new:func([keys], [values])
    The main constructor; same as [:] literal
    [keys] An array of keys to add
    [values] An array of values to add
    This method, if provided with no arguments, will return an empty table. If two arguments are provided, they are interpreted to be an array of keys and an array of values respectively. In that case, a new table will be created by matching/building key-value pairs; keys.count == values.count must be true.
Prototype Methods
  • put:func(key, value)
    Inserts a value into the table
    key The key to use
    value The value to insert
    Hashes key and puts value in the table at the proper location
  • get:func(key)
    Gets a value from the table
    key The key to use
    Hashes key and returns a value if found, otherwise returns nil.
  • hasKey:func(key)
    Returns yes if the table contains a given key
    key The key to look for
  • hasValue:func(value)
    Returns yes if the table contains a given value
    value The value to look for
    This is an expensive operation.
  • keys:func()
    Returns an array of keys
  • values:func()
    Returns an array of values
  • remove:func(key)
    Removes the key-value pair with a given key
    key The key to remove
  • removeValue:func(value)
    Removes all key-value pairs with a given value
    value The value to remove
    This is a more costly operation than remove. All values that match value in the table will be removed.
  • addAll:func(other)
    Adds all the elements from another table
    other The table whose elements you wish to add
    Copies all of the key-value pairs from other to self, modifying self while leaving other unmodified.
  • op_get_index_:func(key)
    Syntactic sugar around []; see get above
    key The key to get
  • op_set_index_:func(key, value)
    Syntactic sugar for [] =; see put above
    key The key to use
    value The value to insert
