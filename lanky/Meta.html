  • versionTag:The tagline that accompanies the current version
  • copyright:The standard copyright tag
Static Methods
  • exec:func(codestr)
    Compiles and executes the given string
    codestr The string to execute
    This function will compile codestr and execute it in the current interpreter context.
  • repl:func()
    Launches the standard REPL
    The REPL that runs under this method is the same as the one used when the interpreter is run without file arguments. When the session ends (i.e. with the user typing Ctrl^d), control is returned to the program.
  • examine:func(obj)
    Provides an overview of the given object
    obj The object to examine
    In most cases, this will just give cursory information. If a function is supplied, this method will decompile and print the opcodes in a pleasing fashion. Useful for debugging and learning about the interpreter.
  • addressOf:func(obj)
    Returns the address of the given object
    obj The object whose address we wish to retrieve
    This method will return a string representing the address of the given object in hexadecimal form
  • allowIntTags:func(allow)
    Sets whether or not to allow tagged integers
    allow Boolean value
    If allow is set to yes, the interpreter will be allowed to use tagged integers in place of actual objects (this is the default behavior). If no, all new integers created will be full-fledged objects.
  • clear:func()
    Clears the terminal screen
  • timout:func(seconds, callback)
    Executes a function after a specific number of seconds
    seconds The number of seconds to timeout
    callback The function to execute
    This method asynchronously waits in the background. Once the current call stack is empty and the seconds have elapsed, the function will be invoked.
  • audit:func()
    Prints the size in bytes of the various C object structs
    Used exclusively for debugging purposes
